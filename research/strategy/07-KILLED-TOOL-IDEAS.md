## Patterns That Kill Tool Ideas

1. **"Free and good enough" is the #1 killer.** In 2026, free AI tools exist for almost every "X generator" category. Canva, Google, Adobe give away what indie devs charge for.
2. **AI wrappers die fastest.** If ChatGPT/Claude can do it with a paste + prompt, a dedicated tool adds marginal value. Only viable if the tool adds processing, formatting, or compliance that raw AI can't.
3. **Image tools are the most saturated category.** Every image manipulation task (generate, edit, remove bg, restore, upscale) has 10-30+ competitors. Avoid unless you have a truly unique angle.
4. **Developer tools trend to free/OSS.** Developers expect tools to be free or open-source. Charging for dev tools requires enterprise sales, not utility tool economics.
5. **Subscription incumbents block pay-per-use entrants.** When established tools charge $15-99/mo, they can absorb low-volume users. Pay-per-use only works if the incumbent is dramatically overpriced for the use case.
6. **Platform features kill indie tools.** When Google, Apple, Adobe, or Canva add a feature, indie tools in that space die. Track platform roadmaps before building.
7. **The AI wave made building trivial — competition is the real problem.** Any solo dev can build any "X generator" in a weekend. The bottleneck is distribution and differentiation, not engineering.
